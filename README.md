# Monte Carlo Integration — f(x) = x²

Обчислення визначеного інтеграла функції f(x) = x² на відрізку [0, 2] методом Монте-Карло з порівнянням аналітичного результату та SciPy quad.

## Запуск

```bash
python monte_carlo.py
```

## Метод Монте-Карло — принцип

Генерується N випадкових точок рівномірно в прямокутнику [A, B] × [0, f(B)].
Частка точок, що потрапили під криву, множиться на площу прямокутника:

```
Інтеграл ≈ (B - A) × f(B) × (кількість точок під кривою / N)
```

## Результати (N = 100 000 точок)

| Метод | Результат | Похибка |
|---|---|---|
| Монте-Карло | 2.640720 | ~0.97% |
| Аналітично (x³/3) | 2.666667 | — |
| SciPy quad | 2.666667 | 2.96e-14 |

Аналітичний розрахунок: ∫₀² x² dx = [x³/3]₀² = 8/3 ≈ 2.6667

## Висновки

Метод Монте-Карло дав результат **2.6407** при точному значенні **2.6667**, відносна похибка склала **~0.97%** при 100 000 точках.

**Точність методу залежить від кількості точок:** зі збільшенням N похибка зменшується пропорційно 1/√N (закон великих чисел). Наприклад:

| N | Очікувана похибка |
|---|---|
| 1 000 | ~3% |
| 100 000 | ~0.3% |
| 10 000 000 | ~0.03% |

**SciPy quad** повертає майже точний результат з похибкою порядку 10⁻¹⁴ — це детермінований метод чисельного інтегрування (квадратура Гауса), який значно точніший за Монте-Карло для одновимірних інтегралів.

**Де Монте-Карло є кращим вибором:**
- Багатовимірні інтеграли (висока розмірність простору), де квадратурні методи стають обчислювально надто важкими.
- Задачі з нерегулярними областями інтегрування.
- Симуляції фізичних процесів (тому метод і отримав свою назву).

Для простих одновимірних інтегралів SciPy quad є точнішим і швидшим рішенням.
